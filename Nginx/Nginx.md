# Nginx

**高性能的HTTP和反向代理服务器**

## 主要功能

反向代理 负载均衡 动静分离

## Web服务器

### 正向代理

代理客户端

正向代理（Forward Proxy）是一种位于客户端与目标服务器之间的中间服务器，核心作用是「代表客户端向目标服务器发起请求」—— 客户端的所有网络请求都先发送给正向代理，再由代理转发到目标服务器，最终将服务器的响应返回给客户端。

#### 典型使用场景

1. 访问被限制的网站：
    - 客户端所在网络（如公司内网、校园网）禁止访问某些外网资源（如特定网站、Docker Hub），通过正向代理  （能访问外网）转发请求，实现间接访问；
    - 跨地区访问（如国内客户端访问国外网站），通过部署在目标地区的正向代理突破地域限制。

### 反向代理

代理服务器端

反向代理（Reverse Proxy）是一种位于目标服务器集群与客户端之间的中间服务器，核心作用是「代表服务器接收客户端的请求」—— 客户端的所有请求都直接发送给反向代理，代理再根据预设规则（如负载均衡、路径匹配）将请求转发到后端的真实服务器，最终将服务器的响应返回给客户端。

#### 典型使用场景

1. 负载均衡（最核心场景）：
    当后端有多台服务器（如部署了相同的 Web 应用、API 服务），反向代理可将客户端请求「均匀分配」到不同服务器，避免单台服务器压力过大
